FeatureModelJson
Generators
noreferences
@@description
<p>Generates a JSON representation of the feature model defined by require statements and mixset features in an Umple file.</p>

<p>Can be invoked using the command line option <code>-g FeatureModelJson</code> or the directive <code>generate FeatureModelJson;</code> in an Umple file.</p>

<p>If the Umple file does not define any feature model, the output will contain <code>{"featureModel":null}</code>.</p>

<p>When the model has an attached <code>UmpleModel</code> with requirements, the output also carries a top-level <code>requirements[]</code> array (sibling of <code>nodes[]</code> and <code>links[]</code>) listing every <code>Requirement</code>. Each entry carries the user-story fields (<code>identifier</code>, <code>language</code>, <code>statement</code>, <code>who</code>, <code>when</code>, <code>what</code>, <code>why</code>) plus nested <code>useCaseSteps[]</code> (with <code>id</code>, <code>stepType</code>, and <code>content</code>) and <code>qualityClasses[]</code> (with <code>name</code> and <code>content</code>). The array is sorted deterministically by <code>identifier</code> so <code>toJSON</code> / <code>fromJSON</code> round-trips remain byte-identical. The <code>requirements</code> key is omitted entirely when no <code>UmpleModel</code> is attached.</p>

<p>Each <code>FeatureLeaf</code> whose <code>mixsetOrFileNode</code> is a <code>Mixset</code> also carries a <code>reqImplementations[]</code> sub-array describing every <code>implementsReq</code> binding owned by that mixset. Each entry records <code>requirementIdentifier</code>, <code>qualityClassName</code> (nullable), <code>implementingFeature</code> (nullable), plus two recomputed-on-serialization booleans <code>requirementResolved</code> and <code>qualityClassResolved</code> that surface whether the parser post-pass successfully bound the entry to its <code>Requirement</code> and <code>QualityClass</code>. The key is emitted only on Mixset leaves — never on <code>UmpleFile</code> leaves nor on <code>FragmentFeatureLeaf</code> nodes.</p>
